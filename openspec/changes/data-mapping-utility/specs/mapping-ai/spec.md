## ADDED Requirements

### Requirement: AI generates CEL mapping expressions from sample data
The system SHALL use Claude (Anthropic API) to analyze source file schema and sample data against a target canonical schema and generate CEL expressions for each target field.

#### Scenario: Generate mappings for NAV file
- **WHEN** the AI generator receives a source schema with fields `["Fund_ID", "Val_Date", "Net_Assets"]`, 5 sample rows, and target schema `dataNav` with fields `["account", "valuationDate", "totalNetAssets"]`
- **THEN** it returns a CEL expression per target field (e.g., `src.Fund_ID.trim()` for account), a confidence score (HIGH/MEDIUM/LOW), and an explanation

#### Scenario: AI uses lookup tables when available
- **WHEN** available lookup tables include `xrefAccount` and the source has an `accountId` column
- **THEN** the AI generates expressions using `lookup('xrefAccount', src.accountId, 'canonicalField')` where appropriate

### Requirement: Each AI-generated expression includes confidence score
Every generated CEL expression SHALL include a confidence level: HIGH (direct mapping or simple transform), MEDIUM (type coercion or format parsing needed), LOW (ambiguous mapping or assumption required).

#### Scenario: High confidence for direct name match
- **WHEN** source field `fund_id` maps to target field `account` with matching sample values
- **THEN** confidence is HIGH

#### Scenario: Low confidence for ambiguous mapping
- **WHEN** multiple source fields could map to a target field or the data patterns are unclear
- **THEN** confidence is LOW with an explanation of the ambiguity

### Requirement: AI-generated expressions are validated before returning
Every CEL expression generated by the AI SHALL be validated by compiling it against the source schema in the cel-python environment before being returned to the caller.

#### Scenario: Invalid expression is regenerated
- **WHEN** the AI generates a CEL expression that fails to compile (e.g., uses a non-existent function)
- **THEN** the system retries generation with error feedback, up to 2 retries

#### Scenario: All expressions pass validation
- **WHEN** the AI generates valid CEL expressions for all target fields
- **THEN** each expression in the response includes `validated: true`

### Requirement: CEL suggestion for individual fields
The API SHALL support suggesting a CEL expression for a single target field given source schema, sample data, and optional context (other field mappings already defined).

#### Scenario: Suggest expression for a date field
- **WHEN** POST `/api/v1/cel/suggest` is called with target field `valuationDate` (type DATE), source schema containing `Val_Date` with sample values `["03/09/2026", "03/10/2026"]`
- **THEN** the response suggests `parseDate(src.Val_Date, 'MM/dd/yyyy')` with HIGH confidence

### Requirement: AI prompt includes function library
The AI generation prompt SHALL include the complete list of available custom CEL functions with their signatures and descriptions so that generated expressions only use registered functions.

#### Scenario: Generated expressions use only registered functions
- **WHEN** the AI generates CEL expressions
- **THEN** every function call in the expression references a function that exists in the cel-python environment (standard CEL + custom extensions)

### Requirement: Structured JSON output from AI
The AI generator SHALL return results as structured JSON with fields: targetField, cel, confidence, explanation, assumptions, and validated.

#### Scenario: Structured response format
- **WHEN** AI generation completes for 5 target fields
- **THEN** the response is a JSON array of 5 objects, each containing `targetField`, `cel`, `confidence`, `explanation`, `assumptions` (array of strings), and `validated` (boolean)
